## 协作约定

### 沟通规则
1. **API 接口变更**: 需要在 COLLABORATION.md 中记录并通知所有相关开发者
2. **架构决策**: 需要在 docs/DECISIONS.md 中记录
3. **任务依赖**: 完成任务前需检查是否有其他开发者的依赖任务

### 代码规范
1. **提交前检查**: 运行 `pytest tests/` 确保测试通过
2. **冲突检测**: 运行 `vibecollab dev conflicts` 检查潜在冲突
3. **上下文同步**: 每次对话结束运行 `vibecollab dev sync`

### 文档维护
1. **个人 CONTEXT.md**: 由各开发者自行维护
2. **全局 CONTEXT.md**: 由系统自动聚合，不要手动编辑
3. **COLLABORATION.md**: 由团队共同维护，记录协作关系
