# VibeCollab v0.12.0 CLI 最佳实践迁移指南

## 概述

v0.12.0 引入了 CLI 驱动的 Workflow 执行模型，将原本 PlanRunner 不支持的 `role_switch`/`check` 等自定义 action 迁移为标准的 `action: cli`。

## 核心变化

### 1. CLI 原子命令增强

新增/增强了以下 CLI 命令：

```bash
# 角色上下文管理
vibecollab role context --export ctx.json    # 导出上下文
vibecollab role context --import ctx.json    # 验证上下文
vibecollab role context --json               # JSON 输出
```

### 2. Workflow 格式变更

**旧格式 (v1.0)**：
```yaml
- action: role_switch
  role: DEV
  plan: "path/to/plan.yaml"
  
- action: check
  check_type: acceptance
```

**新格式 (v2.0 - CLI 驱动)**：
```yaml
- action: cli
  command: "vibecollab role switch DEV"
  expect: { exit_code: 0 }
  
- action: cli
  command: "vibecollab plan run path/to/plan.yaml"
  expect: { exit_code: 0 }
  
- action: cli
  command: "vibecollab check"
  expect: 
    exit_code: 0
    stdout_contains: "All checks passed"
```

### 3. Micro Plan 格式

使用 `action: prompt` 的 micro plan 需要添加 `host` 配置：

```yaml
name: "步骤名称"
description: "步骤描述"
host: file_exchange  # 新增：支持 prompt action

steps:
  - action: prompt
    content: "..."
```

## 迁移步骤

### 步骤1: 更新 Workflow 文件

将所有 `action: role_switch` 改为 `action: cli`：

```yaml
# 之前
- action: role_switch
  role: DESIGN
  plan: ".vibecollab/plans/agents/02-designer/step-01.yaml"

# 之后
- action: cli
  command: "vibecollab role switch DESIGN"
  expect: { exit_code: 0 }

- action: cli
  command: "vibecollab plan run .vibecollab/plans/agents/02-designer/step-01.yaml"
  expect: { exit_code: 0 }
```

### 步骤2: 更新 Micro Plan 文件

为所有使用 `action: prompt` 的 plan 添加 `host: file_exchange`：

```yaml
name: "需求分析"
description: "分析需求"
host: file_exchange  # 添加这一行

steps:
  - action: prompt
    content: "..."
```

### 步骤3: 创建 Shell 脚本 (可选)

对于复杂 workflow，推荐使用 shell 脚本封装：

```bash
#!/bin/bash
set -e

vibecollab role switch PRODUCER
vibecollab plan run .vibecollab/plans/agents/00-producer/step-01.yaml

vibecollab role switch DESIGN
vibecollab plan run .vibecollab/plans/agents/02-designer/step-01.yaml

vibecollab role sync
vibecollab check
```

## 执行方式对比

| 方式 | 命令 | 适用场景 |
|-----|------|---------|
| Workflow YAML | `vibecollab plan run workflow.yaml` | 标准流程，可追踪 |
| Shell 脚本 | `bash scripts/feature-dev.sh` | 复杂流程，灵活组合 |
| 手动 CLI | 逐条执行 CLI | 调试，探索性开发 |

## 常见问题

### Q1: `action: prompt` 报错 "No host adapter configured"

**原因**: Micro plan 使用 `action: prompt` 但没有配置 `host`。

**解决**: 在 plan 文件头部添加 `host: file_exchange`。

### Q2: 如何传递变量给 plan？

使用 `--var` 参数：

```bash
vibecollab plan run workflow.yaml --var feature_name="存档系统" --var priority=P1
```

### Q3: 复杂 workflow 太长，如何简化？

使用 shell 脚本封装，workflow YAML 只保留一个步骤调用脚本：

```yaml
steps:
  - action: cli
    command: "bash .vibecollab/scripts/feature-dev.sh"
    expect: { exit_code: 0 }
```

## 向后兼容性

- `action: cli`, `action: assert`, `action: wait`, `action: loop` 保持不变
- `action: prompt` 需要配合 `host` 配置使用
- 废弃的 `action: role_switch`, `action: check`, `action: decision` 需要迁移
