   - Ensure your AI tool can access `CONTRIBUTING_AI.md`
   - Configure your tool to recognize the VibeCollab protocol

## Agent ID Reference

### Claude (Anthropic)
- Format: Usually auto-generated by Claude's system
- Location: Embedded in Claude's session metadata

### Cursor
- Format: Usually `cursor-agent` or custom ID
- Location: Cursor's agent configuration

### Cline (VS Code)
- Format: Usually `cline-agent` or custom ID
- Location: Cline's agent configuration

### OpenClaw
- Format: `main` or custom agent ID
- Location: `~/.openclaw/openclaw.json` configuration
- Detection: Use `openclaw status` or check session metadata

## Developer Onboarding

### First-time contributors:

1. **Read the protocol:**
   - `CONTRIBUTING_AI.md` - AI collaboration rules
   - `docs/CONTEXT.md` - Current project state
   - `docs/DECISIONS.md` - Important decisions

2. **Choose your tasks:**
   - Check `docs/ROADMAP.md` for planned milestones
   - Use MCP tool `roadmap_status` to see progress
   - Pick a task that matches your skills

3. **Start collaborating:**
   - Use your AI tool's onboarding process
   - For OpenClaw: The `.openclaw/SKILL.md` file provides context
   - For Claude: The `.claude/CLAUDE.md` file provides context
   - For Cursor/Cline: Similar skill files in respective directories

4. **Follow the workflow:**
   - Start: `onboard` (or read context files)
   - Work: Implement your changes
   - End: `check` and `session_save` (if available)

## Protocol Compliance

All registered developers and their AI agents must:

1. **Follow `CONTRIBUTING_AI.md`**
2. **Use MCP tools when available**
3. **Document decisions in `docs/DECISIONS.md`**
4. **Update `docs/CONTEXT.md` after major changes**
5. **Keep this registry updated**
